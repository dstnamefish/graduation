### rem 和 px 区别

`#前端样式` `2025-07-15`

- rem 和 px 是前端开发中常用的长度单位。
- **px（像素）**：绝对单位，表示屏幕上的一个物理像素点，尺寸固定，适合精确布局，但不利于响应式设计。
- **rem（root em）**：相对单位，基于根元素（html）的 font-size，随根元素字体大小变化，适合响应式和移动端开发，便于全局缩放和适配。
- **应用场景**：移动端和响应式页面推荐用 rem，需精确像素控制时用 px。
- **示例**：

  ```css
  html {
    font-size: 16px;
  }
  .box1 {
    width: 32px;
  } /* px 固定宽度 */
  .box2 {
    width: 2rem;
  } /* rem 随 html 字体大小变化 */
  ```

### props 定义与使用

`#Vue3` `#props` `#组件通信` `#父子组件` `#类型校验` `#数据传递`

- 什么是组件的 props ？

  想象你是一个快递员（父组件），你要把包裹（数据）送到客户家（子组件）。

  你在包裹上写明了收件人姓名、地址、电话等信息（这些就是 props）。

  客户（子组件）收到包裹后，可以根据包裹上的信息来处理自己的事情，比如确认收件人、打电话联系等。

- 基本用法

  1.父组件传递数据给子组件

  父组件通过标签属性的方式，把数据传递给子组件

  ```vue
  <!-- 父组件 Parent.vue -->
  <template>
    <ChildComponent :msg="parentMsg" />
  </template>

  <script setup>
  import ChildComponent from './ChildComponent.vue'
  const parentMsg = 'Hello from parent!'
  </script>
  ```

  2.子组件通过 props 接收数据

  子组件用 defineProps（组合式 API）或 props 选项（选项式 API）声明要接收的属性。

  ```vue
  <!-- 子组件 ChildComponent.vue -->
  <template>
    <div>{{ msg }}</div>
  </template>

  <script setup>
  defineProps({
    msg: String
  })
  </script>
  ```

  3.效果

  页面会显示：

  ```text
  Hello from parent!
  ```

### defineProps 在组合式 API 中的优势

`#响应式` `#Vue3` `Props传递` `类型推断`

defineProps 是 Vue 3 组合式 API 中用于声明和获取组件 props 的宏函数。相比传统的 props 选项，defineProps 具有以下优势：

- **类型推断更强**：支持 TypeScript 自动推断 props 类型，无需手动声明类型。
- **代码更简洁**：直接在 setup 顶层声明，减少样板代码。
- **更贴合组合式 API 思路**：与 setup、defineEmits 等 API 配合更自然。
- **更好的 IDE 支持**：智能提示、类型检查更完善。
- **更易于重构和维护**：props 定义与使用更集中，便于阅读和维护。

**示例：**

```vue
<script setup lang="ts">
interface Props {
  title: string;
  count?: number;
}
const props = defineProps<Props>();
</script>
```

> defineProps 只能在 `<script setup>` 中使用，适合类型安全和高规范项目开发。
> defineProps 在编译时转换为标准的 props 选项
