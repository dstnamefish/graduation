# let & const commands
## 名词介绍
1. var：在引入let & const之前，JS中只有var一种声明变量的方法。
2. let：let是ES6新增的声明变量的方法，let声明的变量只在let命令所在的代码块内有效。
3. const：const声明一个只读的常量。一旦声明，常量的值就不能改变。
4. 暂时性死区：在JavaScript中，使用let或const声明的变量会进入暂时性死区(TDZ)，即从代码块({})开始到变量声明语句执行之前的这段区域。在这段区域内访问该变量会抛出ReferenceError。暂时性死区的主要目的是为了避免变量在声明之前被访问，从而防止变量污染和错误。
   注意：
   - 暂时性死区只对let和const声明的变量有效，var声明的变量不会进入暂时性死区。
5. 作用域：定义变量和函数的可访问的范围，控制变量的生命周期和可见性。作用域的最大作用是隔离变量，防止变量污染。
   作用域分为全局作用域和函数作用域，在ES6中，新增了块级作用域。
   - 全局作用域：在整个脚本或模块中都可以访问的变量和函数。最外层的函数和变量拥有全局作用域。所有未定义直接赋值的变量自动声明为全局变量。所有window对象的属性拥有全局作用域。
   - 函数作用域：在函数内部声明的变量和函数，只能在函数内部访问。作用域是分层的，内层作用域可以访问外层作用域的变量，反之则不行。
   - 块级作用域：在代码块（如if语句、for循环等）内部声明的变量和函数，只能在代码块内部访问。
6. 声明: 创建变量、函数、类或其他标识符的语法结构。声明的作用告诉程序这些标识符的存在，并为其分配内存或赋予行为。
7. 作用域链：当访问一个变量时，JavaScript引擎会从当前作用域开始，沿着作用域链向上查找，直到找到该变量或者到达全局作用域。当前作用域->父级作用域->全局作用域，用于查找变量（var/const/let）,由函数的嵌套关系决定。查找失败返回ReferenceError。
8. 立即执行函数表达式(IIFE)：立即执行函数表达式是一种在定义后立即执行的函数。
   立即执行函数表达式的语法：
   ```js
   (function() {
       // 函数体
   })();
   //(function(){}): 最外边的括号将函数声明转换函数表达式
   //(): 第二个括号表示立即执行这个函数表达式，也可以传递参数
   ```
   立即执行函数表达式的作用：
   - 避免全局变量污染：立即执行函数表达式中的变量不会污染全局作用域。
   - 立即执行函数表达式必须用括号括起来，否则会被当作函数声明。
   - 在创建项目后他只能执行一次，不能重复执行。
   - 箭头函数写法：
   ```js
   (() => {
       // 函数体
   })();
   ```
